# Leviathan Sandbox Architecture

The project is structured into four main components:

## 1. Core Logic (`leviathan_sandbox/core/`)

This contains the game engine and physics.

- `game.py`: The main `Game` class, managing the tick loop, entity updates, and state serialization.
- `agent.py`: Defines the `Agent` base class and various implementations (`RandomAgent`, `ScriptedAgent`, `VolcAgent`, `AggressiveAgent`, `SiegeAgent`).
- `objects/`: Contains definitions for game entities (units, buildings).
  - `base_entity.py`: Base class for all entities.
  - `unit/`: Mobile units logic (move, attack).
  - `building/`: Static structures logic.

## 2. CLI Interface (`leviathan_sandbox/cli/`)

- `main.py`: The entry point for running simulations using `typer`.
  - Commands: `fight`, `init`.
  - Responsible for loading strategy files (`.yaml`), initializing agents, running the game loop, and saving replays.

## 3. Web Frontend (`leviathan_sandbox/web/`)

A standalone HTML5 Canvas viewer for replaying game logs.

- `index.html`: The single-page application that loads `.json` replay files and renders the game state using `requestAnimationFrame` for smooth interpolation.
- `assets/`: Static sprites (legacy).
- `assets_animated/`: Animated spritesheets generated by the asset pipeline.

## 4. Asset Pipeline (`tools/asset_pipeline/`)

Automated toolchain for generating game assets using AI.

- `pipeline.py`: Main script that orchestrates:
  1. Text-to-Image (Seedream) -> Base Sprite
  2. Image-to-Video (Seedance) -> Animation Video
  3. Video-to-SpriteSheet (OpenCV + rembg) -> Game-ready Assets
  4. Variant Generation (Blue/Red tinting and flipping)
- `config.yaml`: Defines prompts for each unit type.

## Data Flow

1. **Strategy Definition**: Users write YAML strategy files defining agent behavior/personality.
2. **Simulation**: CLI runs the game loop (headless), where agents make decisions based on `GameState`.
3. **Replay Generation**: The game records all state changes into a JSON file.
4. **Visualization**: The web frontend loads the JSON and renders the battle using the assets generated by the pipeline.
